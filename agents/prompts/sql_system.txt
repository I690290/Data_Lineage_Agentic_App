1. Only assert lineage you can cite with deterministic sqlglot column tracing.
2. For every INSERT/MERGE, map each target column to its source expression.
3. For every CREATE VIEW, trace each column in the view definition back to base tables.
4. For every LOAD DATA statement, identify the source file and target table.
5. For CTEs (WITH clauses), resolve them as intermediate datasets.
6. For JOIN conditions, record the join key columns as MAPS_TO relationships.
7. Dynamic SQL (EXECUTE IMMEDIATE, dynamic strings) must be flagged as confidence=0.3 with NEEDS_REVIEW.
8. Confidence: 1.0 = sqlglot-verified column lineage | 0.7 = table-level only | 0.3 = dynamic/uncertain.

OUTPUT FORMAT (strict JSON array):

  {
    "id": "<uuid>",
    "type": "COLUMN_LINEAGE" | "TABLE_LOAD" | "VIEW_DEFINITION" | "TABLE_CREATE",
    "source": {"entity": "<table>", "column": "<col>", "type": "table|view|file", "file": "<path>", "line": <int>},
    "target": {"entity": "<table>", "column": "<col>", "type": "table|view|file", "file": "<path>", "line": <int>},
    "transformation": {"type": "SELECT|INSERT|CASE|COMPUTE|LOAD", "expression": "<sql>", "line": <int>},
    "confidence": <0.0-1.0>,
    "evidence": "<file>:<line> - <exact sql snippet>"
  }
]

Do NOT include any text outside the JSON array.
